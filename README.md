**Спринт 6, Объектная модель страницы**  
**Введение в проект**   
«Яндекс Самокат» — это сервис для аренды самокатов в Москве и Московской области. Приложение создано специально для отработки навыков студентов «Практикума». Автотесты работают на базе pytest и Selenium и построены в соответствии с объектной моделью страницы. Отчет о тестировании последовательно происходит с помощью платформера Allure и библиотеки allure-pytest.

**Структура репозитория**   
Корневая директория проекта содержит набор тестов и файлы со вспомогательными инструментами:

директория page_object содержит файлы с классами для каждой тестируемой страницы;  
в директории тестов оставлены файлы с тестами, для каждой страницы — свой файл;   
в файлах каталогов локаторы Управляйте локаторами, которые используются в тестах;  
файл conftest содержит фикстуры, необходимые для запуска тестов;  
данные файла хранят тестовые данные полей ввода;  
директория allure_results содержит JSON-файлы с результатами выполнения тестов для генерации отчета;  
в файле require.txt перечислены все внешние зависимые файлы, выполняющие тесты для удобной установки одной команды;    
файл README - текстовое описание проделанной работы.  
**Покрытие**
Задание проекта не требует полностью закрыть приложение тестами. В ходе внедрения необходимо было проверить выполнение определенных требований:

✓ тесты в файле test_logo_redirect проверить переходы на главной странице сервиса и на главной странице «Дзена» при клике по разным частям логотипа в шапке;

✓ тесты в файле test_main_page проверяют содержательное соответствие текста раскрываемых сообщений в разделе «Вопросы о важном» на главной странице приложения;

✓ тесты в файле test_order_page раскрывают всю флоу позитивного сценария оформления заказа. В тесте через параметризацию передаются два набора резервных данных. Для начала оформления заказа использовались две разные точки зрения.

**Запуск всех тестов**  
Установка зависимостей: pip install -r requirements.txt.

Команда для терминала, запускающая все тесты: pytest -v.

**Отчет о тестировании**   
Allure-отчет в формате веб-страницы независимо от команды allure serve allure_results.
